How to Use EPIC

Get your application configured and deploying through EPIC.

Overview

EPIC (Enterprise Pipeline for Infrastructure & Cloud) is PG&E's CI/CD framework built on Azure DevOps. Applications declare their build, test, and deployment intent in a configuration file, and EPIC handles the rest.

The process is simple:

  1. Add a .pipeline/epic.json file to your GitHub repository
  2. Optionally add a .infra/ folder with Terraform configurations for infrastructure
  3. Onboard your app through the + New App button above
  4. Trigger pipeline runs from the app's detail view

The epic.json File

Create .pipeline/epic.json at the root of your repository. This file has two sections:

  • app — Your application's identity, build configuration, and tooling
  • cloud — Your deployment target (AWS account, region, and any app-type-specific settings)
FieldRequiredDescription
app.appNameYesUnique name for your application in EPIC
app.appTypeYesangular, dotnet, python, java, html, php, or ami
app.codePathNoPath to source code relative to repo root (default: /)
app.runtimeVersionNoRuntime version override — defaults per appType: angular/html 18, dotnet 9.x, python 3.11, java 17, php 8.3
app.scanToolNoCode scan tool (e.g., sonarqube)
app.unitTestToolNoTest runner: jest, pytest, junit, xunit, phpunit
cloud.awsAccountIdYesTarget AWS account ID
cloud.awsRegionYesTarget AWS region (e.g., us-west-2)

Infrastructure (Optional)

If your app requires AWS infrastructure (EC2, S3, CloudFront, Aurora, etc.), add a .infra/ folder at the root of your repository with standard Terraform files:

  • terraform.tf — Backend and provider configuration
  • main.tf — Resource definitions
  • variables.tf — Input variables
  • outputs.tf — Output values
  • terraform.auto.tfvars — Variable values

EPIC will automatically detect the .infra/ folder and enable the Infrastructure Deployment option when triggering runs.
